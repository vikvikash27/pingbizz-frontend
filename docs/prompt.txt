B) Roles (RBAC)
Roles:
1. ADMIN: can do anything
2. MANAGER: can create/read/update almost everything BUT cannot delete directly; must request deletion approval from Admin
3. TECHNICAL_OFFICER: can read/update and manage operational configs but cannot delete
4. SUPPORT_ASSISTANT: can view messages, reply when needed, create “support queries/tickets” and assign to Technical Officer; cannot edit system configs; cannot delete

Implement:
- Role enum
- Authorization middleware: requireAuth + requireRole/requirePermission
- Permission map in code (simple RBAC) + ability to extend later

C) Deletion Approval Workflow (IMPORTANT)
- Manager cannot delete resources directly.
- Instead they create a DeletionRequest with:
  - resourceType (e.g. TEMPLATE, CONTACT, CAMPAIGN, AUTOMATION_FLOW)
  - resourceId
  - reason
  - requestedBy
  - status: PENDING | APPROVED | REJECTED
  - approvedBy, approvedAt, rejectedAt, decisionReason
- Admin endpoints:
  - list pending deletion requests
  - approve request → perform deletion (soft delete preferred)
  - reject request
- Technical Officer and Support Assistant can NEVER request deletions (unless explicitly allowed; default NO).
- Every deletion approval or rejection is written into AuditLog.

D) Support Query / Ticket Escalation
- SUPPORT_ASSISTANT can create a Ticket/Query to escalate to TECHNICAL_OFFICER
Ticket fields:
- id, title, description, priority (LOW/MED/HIGH), status (OPEN/IN_PROGRESS/RESOLVED/CLOSED)
- createdBy (support user)
- assignedTo (technical officer user)
- relatedConversationId (string, optional)
- customerContactId (optional)
- internalNotes (array or separate table)
Actions:
- Support Assistant: create ticket, view own tickets, assign ticket to Technical Officer (or request assignment), add comment
- Technical Officer: view assigned tickets, update status, add technical notes, resolve/close
- Manager/Admin: view all tickets; Admin can reassign anyone

E) Audit Logging
- Record sensitive actions:
  - login, logout
  - user creation/role change
  - deletion requested/approved/rejected
  - ticket status changes
- AuditLog fields: actorId, action, entityType, entityId, metadata JSON, ip, userAgent, createdAt
